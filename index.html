IDPMS

Secure Identity Provisioning for IoT Devices using ZKP and SRP

Overview

IDPMS (Identity Provider Microservice) is a custom-built identity solution specifically designed for the secure authentication of Internet of Things (IoT) devices. It leverages advanced cryptographic techniques, including Zero-Knowledge Proofs (ZKP) for establishing trust and the Secure Remote Password (SRP) protocol for robust user authentication.

Registration Flow

The initial registration process involves the IoT device connecting to its designated service provider. The service provider then interacts with IDPMS to securely obtain the necessary cryptographic parameters for the device.

Key Security Features

  • Zero-Knowledge Proof (ZKP): Enables the establishment of trust without revealing sensitive information.
  • Secure Remote Password (SRP) Protocol: Provides strong password-based authentication resistant to common attacks.
  • Device Key Deletion on Tampering: In the event of a mirroring attempt, the IoT device is programmed to self-destruct its key file, rendering the mirrored device unable to authenticate.
  • Computational Security: Guessing the generated cryptographic keys is computationally infeasible.

Key Rotation

To further enhance security, IDPMS implements a key rotation mechanism. The cryptographic keys issued to devices have a defined expiry period. Upon expiration, devices are required to undergo the registration process again to obtain a new set of secure keys. This limits the window of opportunity for any potential key compromise.
